📖 Instrucciones de Uso

Guía para utilizar el Dashboard de KPIs

1. Navegación del Dashboard

Use las pestañas en la parte superior para navegar entre las diferentes secciones:

  • Dashboard KPIs: Vista general de métricas clave por sprint
  • Evolución Sprint: Seguimiento histórico de incidentes
  • Resumen: Matrices de distribución por prioridad y estado
  • Incidentes: Lista detallada de todos los tickets
  • Instrucciones: Esta sección de ayuda

2. Selección de Sprint

En las secciones Dashboard y Resumen, puede seleccionar el sprint específico usando el selector desplegable.

3. Interpretación de KPIs

KPI Meta Descripción
% Finalizados > 80% Porcentaje de incidentes completamente resueltos
% En Curso < 20% Porcentaje de incidentes actualmente en progreso
Highest Abiertos 0 Cantidad de incidentes de máxima prioridad sin resolver
Backlog < 5 Cantidad de incidentes pendientes de iniciar
Tiempo Highest < 2 días Tiempo promedio de resolución para prioridad Highest
Tiempo High < 5 días Tiempo promedio de resolución para prioridad High
Tasa Reapertura < 5% Porcentaje de incidentes que se reabren
Cobertura Resolución > 90% Porcentaje de incidentes del sprint que se resuelven

4. Estados de Cumplimiento

  • ✓ Cumple - El KPI está dentro de la meta establecida
  • ⚠ Revisar - El KPI requiere atención
  • ⚠ CRÍTICO - El KPI está en estado crítico

5. Actualización de Datos

Los datos se actualizan automáticamente cuando se importa el archivo de Jira. La última actualización se muestra en el encabezado de cada sección.

6. Criterios de Sprint

Importante: Todos los incidentes detectados en sprints anteriores al Sprint 30 se asignan automáticamente al Sprint 30 para mantener la consistencia histórica.
